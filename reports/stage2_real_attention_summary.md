# Stage 2 Summary: Real BERT Attention and Exact-Budget Evaluation

Status: **complete 2026-07-03**. Detailed run history is in
[`docs/experiment_log.md`](../docs/experiment_log.md), from the BERT QKV
extraction sections through the Stage 2C exact-budget protocol.

## Question

Stage 1 showed that cheap value-aware proxies can help on synthetic regimes.
Stage 2 asks:

```text
Does the same local-error story survive real attention maps, where entropy,
value structure, punctuation/sink rows, and output norms are no longer synthetic?
```

Answer: **yes for local attention-output error, but only after fixing the
objective and the evaluation protocol.**

## Extraction Sanity Checks

Model: `bert-base-uncased`, documents sampled from 20 Newsgroups.

The extraction pipeline recomputes `P` and `V` from layer inputs and model
weights, then checks:

```text
max |P_recomputed - P_model| ~= 1e-6
||o - sparse(o)|| = delta * ||mu_R - mu_S|| to float precision
```

This anchors the later analysis to real BERT attention rather than synthetic
or mis-extracted tensors.

## What BERT Changed

Synthetic data hid three real-model structures:

1. **Mixed regimes are natural**: BERT heads cover both sharp and diffuse
   attention. The entropy census shows early-layer diffuse heads and sharper
   mid-layer structure.
2. **Relative error matters**: sink/punctuation rows can have small output
   norms, so absolute local error misses the dangerous rows.
3. **Threshold protocols saturate**: many real heads spend far less than the
   target budget under threshold calibration, making comparable-row filtering
   biased.

## UTC-abs -> UTC-rel -> UTC-rel-hat

The original UTC score was absolute:

```text
UTC_abs(k) = delta(k) * ||mu_R_hat(k) - mu_S(k)||
```

It failed on BERT rows where the denominator of relative error was small.

The first relative fix used retained output norm:

```text
UTC_rel(k) = UTC_abs(k) / (||mu_S(k)|| + eta)
```

This fixed some failures but introduced denominator collapse at small k.

The final local scorer candidate is:

```text
o_hat(k) = (1 - delta(k)) * mu_S(k) + delta(k) * mu_R_hat(k)
UTC_rel_hat(k) = UTC_abs(k) / (||o_hat(k)|| + eta)
```

This aligns the proxy with the restricted oracle's relative-error objective
without scanning dropped V per query row.

## Exact-Budget Protocol

The threshold protocol produced only 718 comparable rows out of 4320, because
many heads saturated. Stage 2C replaces it with exact-budget allocation:

```text
For each method:
  build score(row, k)
  choose the smallest risk threshold that fits under total budget
  spend leftover tokens by that method's own marginal score improvements
```

Every method spends the same total token budget, so every row is comparable.

## Main Result

Full BERT exact-budget sweep:

```text
10 docs x 12 layers x 12 heads x 3 budgets = 4320 rows
```

Gap closed vs fixed-k:

```text
method       mean    median   p10     below fixed
mass         0.071   0.245   -0.980   1709
UTC-abs      0.017   0.208   -1.090   1759
UTC-rel      0.380   0.813   -0.179    505
UTC-rel-hat  0.790   0.838    0.541     21
```

Pairwise checks:

```text
rel-hat <= min(abs, rel): 3539 / 4320 = 0.819
rel-hat <= mass:          4273 / 4320 = 0.989
rel-hat worse than fixed:   21 / 4320
```

Interpretation: rel-hat is not just a fix for selected failures. Under strict
matched budget on all BERT head-budget rows, it is the leading deployable local
scorer candidate.

## Failure Diagnosis: 21 Starvation Rows

The remaining below-fixed cases are not random:

```text
by budget: {8: 18, 16: 2, 32: 1}
by layer: L10/L11 = 15 / 21
k_rel_hat < k_oracle: 21 / 21
```

Mechanism:

```text
tight budget
+ max-risk row score underestimated
+ k_rel_hat smaller than k_oracle
=> worst-row starvation
```

Failure subclasses:

```text
denominator-overestimate starvation: 7
numerator-underestimate starvation:  7
mild score miscalibration:           7
```

This matters because it gives a concrete boundary inside the local objective:
rel-hat is strong overall, but can be overconfident on a stable class of tight
budget rows.

## Caveats

- BERT evidence is still a mechanism study: 10 documents, 128-token truncation,
  and selected budgets.
- The metric is local attention-output relative error, not task loss.
- The oracle remains restricted to the top-k-by-probability family.

## Next Stage

should transfer to causal attention with only the causal support/prefix-sum
